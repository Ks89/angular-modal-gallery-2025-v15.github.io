Plain Gallery

Plain gallery contains a list of clickable thumbnails to open modal gallery. Layouts are configurable and users could also use a custom implementation to draw everything they want.


Live demo



API

To add plain gallery in your application, you simply need to add <ks-plain-gallery></ks-plain-gallery> in your component template file and pass the mandatory inputs id, images.

Then, in your Typescript component you have to init the images array with Image class instances and write the method to open modal gallery.


To customize the plain gallery exploring all possibilities, see the table below to pass your configuration. To listen for plain gallery events, check the Output table.

To try more plain gallery live demos with custom configuration, please take a look at the example section of this website.


Inputs

id
Type: number
A mandatory unique id >= 0 of the current gallery.
images
Type: Image[]
The mandatory array of Image objects, that represent both the thumb gallery and the modal images. In fact, an Image is an object with both thumb and image source paths with other additional configuration parameters. For more info check Image. Please don't use this.images.push(img), when you want to modify this array, instead re-assign it to trigger change detection. For instance: this.images = [...this.images, newImage] to create a new array from the original with a new image at the end.
Attention: if you need to add the same image (same url) multiple times in the same plain gallery, you must append to the url a random but unique query param to prevent caching issues. For more info, please read this issue.
config
Type: PlainLibConfigoptional

Interface to configure plain gallery.

Outputs

clickImage

A number that represents the index of the clicked image.


PlainImage

Interface that represent the optional plain image used to open modal image, clicking on it.

If you don't provide this object, @ks89/angular-modal-gallery will scale down the img path of modal image to display the thumbnail.


img
Type: string

Mandatory source path of the plain image.
description
Type: stringoptional

Optional description text. To ignore this feature, don't add this property.
title
Type: stringoptional

Optional title text. It sets 'title' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
alt
Type: stringoptional

Optional alt text. It sets 'alt' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
arialLabel
Type: stringoptional

Optional aria label text. It sets 'aria-label' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.

PlainGalleryConfig

Interface to configure plain gallery.


strategy
Type: PlainGalleryStrategy

Enum that represents the strategy used to show thumbnails. PlainGalleryStrategy.ROW by default.
layout
Type: PlainGalleryLayout

It can be either LineLayout and GridLayout to configure the thumbnail's layout.

PlainGalleryLayout

Empty interface implemented by both LineLayout and GridLayout.


LineLayout

Class to configure a linear plain gallery.


breakConfig
Type: BreakConfig

Object to configure length and wrap.
justify
Type: string

Represent the flexbox 'justify-content' property. For more info check justify-content's doc at Mozilla
size
Type: Size

Object to change the size of thumbnails. width: 50px and height: auto by default.

GridLayout

Class to configure a grid plain gallery.


breakConfig
Type: BreakConfig

Object to configure length and wrap.
size
Type: Size

Object to change the size of thumbnails. width: 50px and height: auto by default.

BreakConfig

Interface to limit the number of items of the plain gallery or to fill other lines.


length
Type: number

Number of items to show. -1 by default.
wrap
Type: boolean

If true, images will wrap, otherwise not. This property correspond to css 'flex-wrap'. false by default.

AdvancedConfig

Interface to use <a> tags instead of <img> and to customize the css background property for thumbnails in plain gallery.


aTags
Type: boolean

Is true, thumbnails will be <a> tags instead of <img>. false by default.
additionalBackground
Type: string

Value useful to change the css background property. '50% 50%/cover' by default. For more info check background CSS3 property

PlainGalleryStrategy

Enum that represent the strategy used to show thumbnails in plain gallery. It can be one of these enum values:


  • ROW: to show thumbnails in a row horizontally
  • COLUMN: to show thumbnails in a row vertically
  • GRID: to show thumbnails in a grid
  • CUSTOM: to show thumbnails with a custom logic
